# ラウンド型協働

## 要点

- `ラウンド型協働` は、全員の入力が揃ってから要約や次段階へ進む協働設計。
- CoDesignAI では、各 user の発話が集まるまで待ち、揃った時点で facilitator が group-level に synthesize する。
- 個別応答より遅くなるが、公平性、整合性、トレーサビリティを上げやすい。

## 定義または輪郭

- 参加者の状態と発話を shared state として管理し、一定条件が満たされたときだけ AI synthesis や round progression を発火させる協働プロトコル。
- quasi-synchronous であり、完全同期ではないが、孤立した turn の積み重ねにもならない中間形式。

## この wiki で重要な理由

- 複数人と複数 agent が混在する場で、`誰の意見をいつ反映したか` を明確にしやすい。
- race condition の抑制や、group-level memory の維持に向いている。
- LLM Wiki の ingest workflow にも、`source を読み終えてからまとめる` `関連 page をまとめて更新する` というラウンド感覚が応用できる。

## 緊張関係 / 異説

- 公平性は上がるが、自由な会話の流れや即応性は下がりうる。
- 全員の入力待ちが、消極的参加者の沈黙や遅延を別の形で固定化する可能性がある。
- real-time collaboration と比べると、臨場感や spontaneity に限界がある。

## open question

- 発話の量ではなく質の偏りをどう扱うべきか。
