# AI支援型協働設計

## 要点

- `AI支援型協働設計` は、人間同士の議論と AI の補助を組み合わせ、設計や意思決定を反復的に前進させる枠組み。
- CoDesignAI では、AI は単なる回答エンジンではなく、`facilitator` `expert` `prompt parser` `image revision support` として分業している。
- 会話、共有メモリ、外部ツール、生成結果が一連の workflow としてつながっている点が重要。

## 定義または輪郭

- 複数の人間参加者と AI role が同じ shared workspace を通じて相互作用し、議論、要約、可視化、再検討を繰り返しながら設計案や知識を育てる協働形式。
- 生成 AI の価値を `一発生成` ではなく、`人間の協働プロセスを構造化し増幅すること` に置く。

## この wiki で重要な理由

- LLM Wiki の考え方そのものが、`source` `memory` `synthesis` `next action` をつなぐ AI支援型協働設計の一種として読める。
- 創作、研究、企画、コミュニティ運営など、都市デザイン以外へも転用しやすい抽象度を持つ。
- `AI が何を作るか` より `AI をどう役割配置するか` という設計論へ視点をずらせる。

## 緊張関係 / 異説

- AI による効率化は、参加の質や正当性を自動的に保証しない。
- agent 間の役割分担は有効だが、役割が固定されすぎると議論が硬直化する。
- aligned goals を前提にしがちで、利害対立や権力差の扱いはまだ弱い。

## open question

- どの仕事は facilitator に、どの仕事は domain expert に分けるべきか。
- memory を長く保つほど便利になる一方で、どこで圧縮・忘却すべきか。
- UI は chat 中心で十分か、それとも graph / canvas / table を前提にすべきか。
